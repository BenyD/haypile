# Postmortem: API outage, 14 March 2026

Severity: SEV-2. Duration: 47 minutes. Customer impact: elevated error rates
on all write endpoints.

## Summary

A deploy doubled the number of API pods without raising the database
connection pool ceiling. Connection pool exhaustion caused request queuing;
client timeouts then triggered a retry storm that kept the pool saturated
even after traffic normalized.

## Timeline

- 14:02 deploy completes, pod count 12 → 24
- 14:09 p99 latency alarms fire; error rate climbs to 18%
- 14:31 root cause identified: max_connections at 100, pods requesting 240
- 14:49 pool ceiling raised, retries given jittered exponential backoff; recovery

## Action items

1. Connection pool size becomes a deploy-gated config, reviewed with pod count.
2. All internal clients adopt jittered exponential backoff (no naked retries).
3. Add a synthetic canary that exercises the write path continuously.
